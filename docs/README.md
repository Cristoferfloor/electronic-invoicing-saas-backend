# Documentation

This directory contains the project documentation, organized by sprints and plans.

## Directory Structure

```
docs/
├── project_master_plan.md    # The overall master plan for all sprints
└── sprint_1/                 # Documentation specific to Sprint 1
    ├── plan.md               # Detailed plan for Sprint 1
    ├── summary.md            # Executive summary for Sprint 1
    ├── api_endpoints.md      # API documentation for Sprint 1
    └── technical_report.md   # Technical report for Sprint 1
```

## How to use

- Start with `project_master_plan.md` for a high-level overview.
- Navigate to the specific sprint folder (e.g., `sprint_1/`) for detailed documentation on that sprint.
